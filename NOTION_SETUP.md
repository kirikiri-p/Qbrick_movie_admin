# Notion リアルタイム同期 セットアップ手順（超ていねい版）

このサイトでデータ（衣装・小道具など）を保存すると、自動で **2つの Notion データベース**
（衣装メイク一覧／小道具一覧）に反映されるようにします。**サイト → Notion の片方向**です。

作業は大きく3つ。
- **STEP 1**: Notion 側の準備（連携アプリ作成・DB 2つ作成・共有・ID取得）
- **STEP 2**: パソコンの準備（Node.js と Firebase CLI のインストール）
- **STEP 3**: デプロイ（鍵を登録してアップロード）

> ⏱ 目安: 初めてでも 30〜60分。途中で分からなくなったら、そのエラー文を私に貼ってください。

> 💰 料金: Cloud Functions は Firebase の Blaze（従量課金）プランが必須です。
> ただしこの規模なら無料枠に収まり、実際の請求は **¥0** の見込みです（STEP 3-6 で予算アラートも設定）。

# STEP 1: Notion 側の準備

## 1-1. 連携アプリ（インテグレーション）を作る
1. ブラウザで https://www.notion.so/my-integrations を開く（Notionにログイン）
2. **「New integration（新しいインテグレーション）」** をクリック
3. 入力する:
   - Name（名前）: `Qbrick連携` など分かりやすい名前
   - Associated workspace（関連ワークスペース）: 自分のワークスペースを選択
   - Type: **Internal** を選択
4. **「Save」** をクリック
5. 作成後の画面で **「Configure integration settings」**（または Capabilities）を開き、
   **Content Capabilities** で以下にチェックが入っているか確認:
   - ☑ Read content（コンテンツの読み取り）
   - ☑ Update content（コンテンツの更新）
   - ☑ Insert content（コンテンツの挿入）
6. **「Internal Integration Secret」** の欄の「Show」→「Copy」でコピー。
   `secret_xxxxx...` または `ntn_xxxxx...` という長い文字列です。
   
   → これを後で使う **NOTION_TOKEN** としてメモ帳などに控える（他人に見せない鍵です）

## 1-2. データベースを2つ作る
Notion で適当なページを開き、本文に **半角で `/database`** と打って
**「Database - Full page（データベース：フルページ）」** を選ぶと表（DB）が作れます。これを **2つ** 作ります。

> ⚠️ **一番ありがちな間違い: 縦と横が逆**
> 項目名は **横の見出し（列）** に並べます。**縦（行）に項目名を入れるのは間違い**です。
> データ（各衣装）は後でアプリが**1行ずつ自動で**埋めるので、最初は**行は空っぽ**でOK。
> ```
> 正解 → ┌────┬──────────┬──────────┬─────…   ← 横が見出し（列）
>        │名前 │使用シーン番号│衣装セット名│ …
>        ├────┼──────────┼──────────┼─────…
>        │（空）│ （空）     │ （空）    │ …       ← 行は空でOK（自動で埋まる）
> ```

### ① 衣装メイク一覧（DBその1）
作ったら、表の列（プロパティ）を**以下のとおり完全一致**で作ります。
- 列の足し方: 表の一番右の列ヘッダーの **「＋」** をクリック → 名前を入力 → 種類を選ぶ
- 列の種類変更: 列ヘッダーをクリック →「Edit property（プロパティを編集）」→ Type で種類を選ぶ
- 最初からある「Name」列は名前を **「名前」** にリネーム（種類はタイトルのまま）

| プロパティ名（列名） | 種類 |
|---|---|
| 名前 | タイトル（Title） |
| 映画 | テキスト |
| 使用シーン番号 | テキスト（Text） |
| 衣装セット名 | テキスト |
| 誰の衣装か | テキスト |
| 構成衣装 | テキスト |
| それぞれの詳細 | テキスト |
| 使用日 | テキスト |
| ステータス | セレクト（Select） |
| 用意する人 | テキスト |
| 備考 | テキスト |
| 使用金額 | テキスト |
| Key | テキスト |
| MovieKey | テキスト |

### ② 小道具一覧（DBその2）
同じ要領でもう1つ作ります。

| プロパティ名（列名） | 種類 |
|---|---|
| 名前 | タイトル |
| 映画 | テキスト |
| 使用シーン番号 | テキスト |
| 小道具名 | テキスト |
| 詳細 | テキスト |
| 使用日 | テキスト |
| ステータス | セレクト |
| 用意する人 | テキスト |
| 備考 | テキスト |
| 使用金額 | テキスト |
| Key | テキスト |
| MovieKey | テキスト |

> 📝 ポイント
> - 列名は**ひらがな・漢字も含めて1文字も間違えない**でください（合わないとその列が空になります）。
> - `Key` `MovieKey` は内部処理用です。作ったら列ヘッダー →「Hide in view（ビューで非表示）」で隠してOK。
> - **「用意する人」は自動では入りません**（Notion で手入力する欄です。手入力した値は同期で消えません）。
>   小道具の「備考」も同じく手入力用です。
> - 「ステータス」セレクトの選択肢（未着手/準備中/準備完了）は自動で作られるので、事前に作らなくてOK。

## 1-3. 2つのDBを連携アプリに「接続」する（重要・忘れやすい）
各データベースを**フルページで開いた状態**で:
1. 右上の **「•••（三点メニュー）」** をクリック
2. **「Connections（コネクト / 接続）」** → **「Add connections」**
3. 1-1で作った **「Qbrick連携」** を選ぶ →「Confirm」
4. **もう1つのDBでも同じことを行う**（2つとも接続しないと書き込めません）

## 1-4. それぞれのデータベースID を取得する
1. データベースを**フルページで開く**
2. 右上「•••」→ **「Copy link（リンクをコピー）」**
3. コピーされたURLは例えばこんな形:
   ```
   https://www.notion.so/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx?v=yyyyyyyy...
   ```
   この **`?v=` より前の 32桁の英数字**（`xxxx...` の部分）がデータベースIDです。
   （`-` ハイフンが混ざっていてもOK。混ざっていればハイフンごと控えてください）
4. 控える:
   - 衣装メイク一覧のID → **NOTION_COSTUME_DB**
   - 小道具一覧のID → **NOTION_PROP_DB**

> これで STEP 1 完了。手元に **3つの値**（NOTION_TOKEN / NOTION_COSTUME_DB / NOTION_PROP_DB）が
> 揃っているはずです。

# STEP 2: パソコンの準備（Windows 想定）

## 2-1. Node.js を入れる
1. https://nodejs.org/ja を開き、**「LTS」** 版（20系）をダウンロードしてインストール
2. インストール後、**PowerShell** を開いて確認:
   ```powershell
   node -v
   ```
   `v20.x.x` のように表示されればOK。

## 2-2. Firebase CLI を入れる
PowerShell で:
```powershell
npm install -g firebase-tools
```
確認:
```powershell
firebase --version
```
バージョン番号が出ればOK。

# STEP 3: デプロイ（このプロジェクトフォルダで作業）

PowerShell で、このフォルダ（`Qbrick_movie_admin`）に移動してから実行します。
```powershell
cd "C:\Users\tente\Desktop\避難所HTML\Qbrick_movie_admin"
```

## 3-1. Firebase にログイン
```powershell
firebase login
```
ブラウザが開くので、Firebase（＝このアプリ）で使っている Google アカウントで許可。

## 3-2. リージョン（地域）を確認・合わせる
1. Firebase コンソール → 対象プロジェクト → **Firestore Database** を開く
2. 画面に出ている **「ロケーション」**（例: `asia-northeast1`、`nam5` など）を確認
3. `functions/index.js` の先頭にある次の行を、そのロケーションに合わせる:
   ```js
   const REGION = 'asia-northeast1';
   ```
   - 東京: `asia-northeast1`
   - 米国 nam5 の場合: `us-central1`
   ※ここがズレると 3-5 のデプロイで失敗します。分からなければロケーション名を私に教えてください。

## 3-3. 関数の依存をインストール
```powershell
cd functions
npm install
cd ..
```

## 3-4. 3つの鍵（シークレット）を登録
1つずつ実行し、プロンプトが出たら STEP 1 で控えた値を貼り付けて Enter:
```powershell
firebase functions:secrets:set NOTION_TOKEN
firebase functions:secrets:set NOTION_COSTUME_DB
firebase functions:secrets:set NOTION_PROP_DB
```

## 3-5. デプロイ（アップロード）
```powershell
firebase deploy --only functions
```
- 初回は **「Blaze プランへのアップグレードが必要」** と促されます。表示されるリンクから
  Firebase コンソールで **Blaze（従量課金）に変更**（カード登録）してから、もう一度このコマンドを実行。
- `Deploy complete!` と出れば成功です。

## 3-6.（推奨）予算アラートを設定
1. https://console.cloud.google.com/ を開く →（Firebaseと同じプロジェクトを選択）
2. 左メニュー **「お支払い」→「予算とアラート」→「予算を作成」**
3. 金額を `¥1` などにして作成 → 超えそうになるとメールで通知が来ます（安心用）。

# 動作確認
アプリで衣装か小道具を1件編集して**保存** → 数秒〜数十秒後に、Notion の2つのDBに行が
増える/更新されればOK！
（映画ごと削除すると、その映画の行は Notion 側でアーカイブ＝ゴミ箱に移動します）

# うまくいかない時（トラブル対処）

| 症状 | 原因・対処 |
|---|---|
| デプロイで `region` 関連のエラー | 3-2 のリージョンが Firestore と不一致。`functions/index.js` の `REGION` を直す |
| Notion に何も増えない | 1-3 の「接続（Add connections）」を**両方のDB**でやったか確認 |
| 一部の列だけ空になる | 1-2 の**列名が1文字でも違う**と空になる。列名を見直す |
| `401 Unauthorized` 等 | `NOTION_TOKEN` の貼り間違い。3-4 をやり直す（`secret_`/`ntn_` で始まる値か確認） |
| `object_not_found` / DBが見つからない | `NOTION_COSTUME_DB` / `NOTION_PROP_DB` のIDが違う、または接続していない |
| ログを見たい | Firebase コンソール → Functions → `syncMovieToNotion` → 「ログ」で実行結果・エラーが見られる |

エラー文が出たら、その文章をそのまま私に貼ってください。原因を特定して直します。

# メモ（仕様）
- 同期は**サイト→Notionの片方向**です。Notion 側で「用意する人」などを手入力するのはOK
  （同期で消えません）。それ以外の同期対象の列を Notion で手編集しても、次回の保存時に上書きされます。
- 1行 = (誰の, 名称) のまとまり。複数シーンで使う衣装/小道具は「使用シーン番号」「使用日」に
  まとめて表示されます。
- 1回の保存で全行を再同期します（編集が多い日でも無料枠内に収まる想定）。
