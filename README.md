# Qbrick 撮影管理アプリ — リファクタリング版 (v11)

## デプロイ方法
リポジトリのファイルをこの内容でまるごと差し替えてください。
（`Qbrick_icon.png` と `Qbrick_管理テンプレート.xlsx` はそのまま流用します）

```
index.html
style.css
daily_schedule_template.xlsx … 日々スケジュールのテンプレート（※リポジトリ直下に必須）
js/
  callsheet.js… 日々スケジュール（たたき台）の生成
  toast.js    … トースト通知（保存完了などの自動で消える通知）
  main.js      … 起点。イベント結線・ダークモード・Firestore購読
  state.js     … 画面間で共有する状態
  utils.js     … エスケープ・日付正規化などの共通関数
  firebase.js  … Firebase初期化とトランザクション保存
  nav.js       … 画面遷移（ハッシュ書き換え）
  router.js    … ハッシュを解釈して各画面を描画
  home.js      … ホーム（映画一覧・カレンダー）
  movie.js     … 映画画面（シーン一覧・衣装/小道具一覧）
  detail.js    … シーン詳細（閲覧・編集）
  search.js    … シーン検索
  items.js     … 入力部品（日付・衣装枠・小道具枠・サジェスト）
  excel.js     … Excelインポート/エクスポート
```

Firebaseコンソール側の作業は不要です。

## 新機能
1. **一括管理マネージャーが編集者モード限定に**（閲覧モードでは非表示）
2. **「最速使用日が早い順」で並べ替え中、各アイテムに最速使用日を表示**
3. **衣装一覧・小道具一覧の先頭に準備達成率カードを表示**
   （準備完了のアイテム数 ÷ 全アイテム数、プログレスバー付き）
4. **映画一覧に撮影進捗率を表示**
   （撮影済みシーン数 ÷ 全シーン数、プログレスバー付き）

## バグ修正
- **表示崩れ・スクリプト注入対策**: ユーザー入力（シーン名・メモ・衣装名など）を
  画面に出す前にすべてHTMLエスケープするように変更
- **「'」や「"」を含む名前でボタンが壊れるバグを根絶**:
  インラインonclickへの文字列埋め込みをやめ、addEventListenerで結線
- **同時編集の上書き対策**: 保存をトランザクション化し、
  「最新データを読み直してから自分の変更だけ適用」する方式に変更。
  別の人が他のシーンを編集していても互いの変更が消えにくくなった
- **Excelの列ズレ修正**: セル内の区切り文字をカンマ→「|」に変更。
  「1,500円」のような値があっても列がズレない。
  旧形式（カンマ区切り）のファイルも自動判別してそのまま読み込み可能
- **日付の正規化**: 「2026/1/5」「2026年1月5日」等の表記もYYYY-MM-DDに
  統一して保存・照合。カレンダーや日別予定に出てこない問題を解消
- **壊れたデータ1件で全体が止まる問題を修正**: 読み込めないドキュメントは
  スキップして他のデータは表示する
- **ダークモードを端末に記憶**（リロードしても維持）
- **映画削除時に端末に残る参加フラグも掃除**
- 保存・削除の失敗時にエラーを通知（以前は無言で失敗していた）

## 注意点
- キャッシュバスター（?v=7）を更新済み。今後ファイルを更新したら
  index.html内の `?v=` の数字を上げるとキャッシュ問題を防げます
- 保存はオンライン時のみ確実に動作します（トランザクション化の影響で、
  オフライン時は保存失敗のメッセージが出ます）
- 編集者モードのパスワードは従来どおりJS内に書かれています
  （身内ツールとして許容、という方針に基づく）

- iPhoneで入力欄タップ時に画面が勝手にズームする問題を修正（スマホは入力文字16px）
- スマホの全画面詳細パネルを動的ビューポート(100dvh)+セーフエリア対応に
- 撮影済みトグルが保存失敗時に見た目だけ切り替わったままになる問題を修正（自動で巻き戻し）
- 他端末で削除された映画のページを開いていたら自動でホームへ戻る
- 「✕閉じる」「編集をキャンセル」がスクロールに追従するように
- 保存完了などの通知をalertからトーストに変更（OKタップ不要）
- 長い映画タイトルでヘッダーが崩れないよう省略(…)表示に
- カレンダーの絵文字は2つまで・ドットは6個+「+n」表示であふれ防止
- 衣装枠の✕削除ボタンなどのタップ領域を拡大
- シーン番号がテキスト入力になり「13A」「7-2」などの枝番が使用可能に
- 【編集パネルの再設計】基本情報のみ展開した状態で開き、衣装・小道具は
  件数つきアコーディオンに。「保存/キャンセル」は画面下に常時追従表示

## v9での追加機能: 日々スケジュールのたたき台作成
日別画面（カレンダーの日付タップ → 撮影予定）に、編集者モード限定で
「📋 日々スケのたたき台を作成（Excel）」ボタンが出ます。

- テンプレートに書き込んだExcelをその場でダウンロード
- 撮影順序は 時間帯 M(朝)→D(昼)→E(夕)→N(夜) の順、同じ時間帯内は
  同じ場所のシーンをまとめて移動回数を最小化
- 場所が変わる箇所に「移動（A→B）・撮影準備」行を自動挿入
- 「集合・機材詰込み」「昼休憩」「撮影終了」「片付け・解散」の定型行入り
- スタッフ集合時間（初期値8:30）、1シーンの撮影時間（60分）、移動時間（30分）、
  昼休憩の開始目安と長さ、Day番号、集合場所は生成時に調整可能
- 撮影開始/終了予定時間は積み上げ計算で自動記入
- 日の出・日の入は熊本基準の天文計算で自動記入（誤差±1〜2分）
- シフト・キャスト・天候・PAGE・現場住所などアプリにないデータは空欄のまま
- 撮影以外の行（集合・移動・昼休憩・撮影終了・片付け）は時刻と内容が
  グレー文字になり、撮影行が一目で分かる

### 導入時の注意
- 同梱の `daily_schedule_template.xlsx` をリポジトリ直下に置いてください
  （ファイル名はこのまま。アプリがこの名前で取得します）
- Excel書き込みにはExcelJS（CDN）を生成時にだけ読み込みます

## v11での追加機能: 登場人物・配役の登録と日々スケへの自動記入
- 映画の基本情報画面に「登場人物・配役」セクションを追加。
  登場人物名（必須）と役者名（任意）のペアを必要なだけ登録できる
- 日々スケジュール生成時、その映画の登場人物全員を「役名/出演者」欄に
  自動記入（役者名があれば「登場人物（役者）」の形式）
- 部署割（監督/制作/撮影/録音/照明）の枠を左下に自動作成。
  監督名はアプリ登録から自動で記入、他は空欄（実名は現場で記入）
- STAFF連絡先欄にも監督名を自動記入

### 注意
- 部署割は現状「■現場住所」の枠に記入しています。専用枠が必要なら
  テンプレート側にエリアを追加することも可能です

## v12での追加機能: 登場人物をシーンに割り当て＆Excel連携
- **シーンへの登場人物割り当て**: シーンの追加・編集画面に「登場人物」チェック欄を追加。
  基本情報で登録した登場人物の中から、そのシーンに出る人物を選べる。
  シーンカード・シーン詳細に登場人物バッジ（🎭）を表示
- **検索に登場人物を追加**: シーン検索画面で「登場人物」でも絞り込める
- **日々スケジュールに登場人物の○列**: テンプレ中央の空き列（L〜T、最大9人）に
  その日に出る登場人物を縦書き見出しで並べ、各シーン行に出演する人物を○で記入
  （添付イメージの香盤表と同じ形式）。集合場所・機材場所も自動記入（既定: 知命堂）

### テンプレート修正に伴う注意（v12時点）
- daily_schedule_template の列対応: H=L/S, I=D/N, **J=CUT, K=PAGE**, L〜T=登場人物○列
- **ExcelJSはフローティング図形（テキストボックス・四角形）を保持できない**ため、
  テンプレに挿入した図形（「CALL SHEET」「決」「撮影メモ」枠など）はDL時に消える。
  残したい内容は必ず罫線を引いた実セルに置くこと
- 部署割の書き込み先は callsheet.js の `DEPT_CELL` 定数で指定（現状は左下「現場住所」枠 B39）。
  専用枠を実セルで作ったら、その左上セル番地に変更する
- **管理Excelの往復が完全に**: エクスポートに「撮影ステータス・時間帯・メモ・登場人物」
  の列を追加（13列目以降）。エクスポート→再インポートで全情報が復元される。
  登場人物は「名前（役者）」形式で出力し、再インポート時に配役登録も復元。
  先頭12列は従来どおりなので、過去のテンプレ・旧ファイルもそのまま読み込める
- キャッシュバスターを ?v=12 に更新
