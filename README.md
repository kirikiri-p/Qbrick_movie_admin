# Qbrick 撮影管理アプリ — リファクタリング版 (v7)

## デプロイ方法
リポジトリのファイルをこの内容でまるごと差し替えてください。
（`Qbrick_icon.png` と `Qbrick_管理テンプレート.xlsx` はそのまま流用します）

```
index.html
style.css
js/
  main.js      … 起点。イベント結線・ダークモード・Firestore購読
  state.js     … 画面間で共有する状態
  utils.js     … エスケープ・日付正規化などの共通関数
  firebase.js  … Firebase初期化とトランザクション保存
  nav.js       … 画面遷移（ハッシュ書き換え）
  router.js    … ハッシュを解釈して各画面を描画
  home.js      … ホーム（映画一覧・カレンダー）
  movie.js     … 映画画面（シーン一覧・衣装/小道具一覧）
  detail.js    … シーン詳細（閲覧・編集）
  search.js    … シーン検索
  items.js     … 入力部品（日付・衣装枠・小道具枠・サジェスト）
  excel.js     … Excelインポート/エクスポート
```

Firebaseコンソール側の作業は不要です。

## 新機能
1. **一括管理マネージャーが編集者モード限定に**（閲覧モードでは非表示）
2. **「最速使用日が早い順」で並べ替え中、各アイテムに最速使用日を表示**
3. **衣装一覧・小道具一覧の先頭に準備達成率カードを表示**
   （準備完了のアイテム数 ÷ 全アイテム数、プログレスバー付き）
4. **映画一覧に撮影進捗率を表示**
   （撮影済みシーン数 ÷ 全シーン数、プログレスバー付き）

## バグ修正
- **表示崩れ・スクリプト注入対策**: ユーザー入力（シーン名・メモ・衣装名など）を
  画面に出す前にすべてHTMLエスケープするように変更
- **「'」や「"」を含む名前でボタンが壊れるバグを根絶**:
  インラインonclickへの文字列埋め込みをやめ、addEventListenerで結線
- **同時編集の上書き対策**: 保存をトランザクション化し、
  「最新データを読み直してから自分の変更だけ適用」する方式に変更。
  別の人が他のシーンを編集していても互いの変更が消えにくくなった
- **Excelの列ズレ修正**: セル内の区切り文字をカンマ→「|」に変更。
  「1,500円」のような値があっても列がズレない。
  旧形式（カンマ区切り）のファイルも自動判別してそのまま読み込み可能
- **日付の正規化**: 「2026/1/5」「2026年1月5日」等の表記もYYYY-MM-DDに
  統一して保存・照合。カレンダーや日別予定に出てこない問題を解消
- **壊れたデータ1件で全体が止まる問題を修正**: 読み込めないドキュメントは
  スキップして他のデータは表示する
- **ダークモードを端末に記憶**（リロードしても維持）
- **映画削除時に端末に残る参加フラグも掃除**
- 保存・削除の失敗時にエラーを通知（以前は無言で失敗していた）

## 注意点
- キャッシュバスター（?v=7）を更新済み。今後ファイルを更新したら
  index.html内の `?v=` の数字を上げるとキャッシュ問題を防げます
- 保存はオンライン時のみ確実に動作します（トランザクション化の影響で、
  オフライン時は保存失敗のメッセージが出ます）
- 編集者モードのパスワードは従来どおりJS内に書かれています
  （身内ツールとして許容、という方針に基づく）
